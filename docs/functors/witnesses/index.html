An Equivalence that is not an Isomorphism

Name: An Equivalence that is not an Isomorphism

Definition:

Consider the category $\mathbf{FinVect}_k$ of finite-dimensional vector spaces over a field $k$, and let $C$ be the full subcategory consisting of just one representative from each isomorphism class (e.g., $k^n$ for each $n \geq 0$). The inclusion functor $F: C \to \mathbf{FinVect}_k$ is fully faithful and essentially surjective, hence an equivalence. However, it is not surjective on objects, so it is not an isomorphism of categories.

Comments:

This is the canonical example showing that equivalence is weaker than isomorphism.

Name: An Embedding that is not an Isomorphism

Definition:

A Right Adjoint that is not an Equivalence

Name: A Right Adjoint that is not an Equivalence

Definition:

The forgetful functor $U: \mathbf{Grp} \to \mathbf{Set}$ from groups to sets has a left adjoint (the free group functor), making it a right adjoint. However, $U$ is neither full, nor faithful, nor essentially surjective, so it is not an equivalence.

Comments:

Most forgetful functors are adjoints but not equivalences.

A Monadic Functor that is not an Equivalence

Name: A Monadic Functor that is not an Equivalence

Definition:

The forgetful functor $U: \mathbf{Ab} \to \mathbf{Set}$ from abelian groups to sets is monadic (by Beck's monadicity theorem), but it is not an equivalence since it is neither full nor essentially surjective.

Monadic functors create algebras for a monad but need not be equivalences.

A Left Adjoint that does not Preserve all Limits

Name: A Left Adjoint that does not Preserve all Limits

Definition:

The tensor product functor $- \otimes M: \mathbf{Ab} \to \mathbf{Ab}$ for a fixed abelian group $M$ is a left adjoint (its right adjoint is $\text{Hom}(M,-)$). However, tensor product does not preserve all limits. For example, it does not preserve infinite products in general.

An Essentially Injective Functor that is not Injective-on-objects

Name: An Essentially Injective Functor that is not Injective-on-objects

Definition:

Consider a functor between groupoids where two distinct objects in the domain are sent to isomorphic (but distinct) objects in the codomain. For example, the functor from the groupoid with two objects ${A, B}$ and one isomorphism between them, to the groupoid with one object and its identity, sending both $A$ and $B$ to the same object. This functor is essentially injective (if $F(X) \cong F(Y)$, then $X \cong Y$ in the domain), but not injective on objects.

Comments:

Essential injectivity is about isomorphism classes, not equality of objects.
